# Mentira UX Product Review

## Review

- Identify what the user is trying to accomplish and the primary action.
- Check whether the flow is clear from entry to outcome.
- Remove unnecessary steps, hidden decisions, or ambiguous actions.
- Review loading, error, empty, and success states.
- Check labels, helper text, CTAs, feedback, and copy clarity.
- Review visual hierarchy, readability, spacing, and ease of navigation.
- Consider desktop, tablet, mobile, touch interactions, and accessibility basics.
- Call out friction even if the screen looks visually polished.

## Decision Rules

- Prioritize user comfort, clarity, speed of use, and consistency.
- Do not sacrifice usability for decoration.
- Do not propose visual changes that are attractive but unhelpful.
- Do not move data, permission, or backend responsibilities for UX preference alone.
- Coordinate with architecture, content, or database skills when the UX depends on data flow or permissions.

## Output

- Name the UX problem briefly.
- Propose the smallest useful change.
- State which device sizes, roles, or states still need validation.
